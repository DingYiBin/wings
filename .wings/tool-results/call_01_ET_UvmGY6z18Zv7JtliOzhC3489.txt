208	## 2b. routing — API 候选池
209	
210	**位置**: `src/wings/routing/`
211	**依赖**: 无（纯数据结构 + 随机算法）
212	
213	### 设计目标
214	
215	wings 的核心差异点：每次模型调用都从当前任务的 API 候选池中加权随机选择一个 API。用户通过打分和设置调整各任务类型的候选池。
216	
217	### 为什么 routing 先于 models 实现
218	
219	routing 模块零依赖（不依赖 messages、models、tools），可以独立开发和测试。其他模块通过 Protocol 依赖 routing，不依赖具体实现。即便后续彻底重写池的实现（换存储、换算法、换继承逻辑），也不影响调用方。
220	
221	### 分层架构
222	
223	```
224	┌─────────────────────────────┐
225	│  ModelSelector (Protocol)    │  ← 其他模块只依赖这个
226	├─────────────────────────────┤
227	│  APIPoolManager              │  ← 管理池、选择、调整、持久化
228	├─────────────────────────────┤
229	│  weighted_select()           │  ← 纯函数：列表 → 选中项
230	│  resolve_parent()            │  ← 纯函数：task_type → parent
231	│  resolve_pool()              │  ← 纯函数：task_type → 最终池
232	├─────────────────────────────┤
233	│  PoolEntry, TaskPool         │  ← Pydantic 数据结构
234	│  PoolConfig                  │  ← 持久化格式
235	│  TASK_HIERARCHY              │  ← 静态继承链（单一真相源）
236	└─────────────────────────────┘
237	```
238	
239	纯函数层（selector + tasks）无状态、无副作用，可独立测试和替换。有状态的管理器层（APIPoolManager）只做编排和锁管理。Protocol 层让调用方与实现解耦。
240	
241	### ModelSelector Protocol
242	
243	其他模块只依赖这个 Protocol，不依赖 `APIPoolManager` 具体类。
244	
245	```python
246	# protocol.py
247	from typing import Protocol
248	
249	class ModelSelector(Protocol):
250	    """选择模型的接口。AgentLoop、QueryEngine 只依赖此协议。"""
251	
252	    def select(self, task_type: str, override: str | None = None) -> str:
253	        """返回选中的 api_id。
254	
255	        Args:
256	            task_type: 任务类型 ("main", "subagent/explore", "skill/commit", ...)
257	            override: session 级模型锁定（用户 /model 命令设置），绕过池选择。
258	
259	        Returns:
260	            api_id，如 "anthropic/claude-opus-4-6"
261	
262	        Raises:
263	            NoAPIAvailable: 池中无可用 API
264	        """
265	        ...
266	```
267	
268	### override 语义
269	
270	| 来源 | 持久性 | 说明 |
271	|------|--------|------|
272	| `/model <api>` | session 级锁定 | 后续所有 turn 使用此模型，直到 `/model random` |
273	| `/model random` | — | 清除锁定，恢复池选择 |
274	| Skill `model` 字段 | per-call | 该 skill 的一次调用使用指定模型，不影响其他 turn |
275	
276	实现上，`override` 是 session 级持久锁定值，由 AgentContext 携带。Skill 的 per-call 覆盖在子 agent 创建时处理（不经过池的 select）。

278	### 数据结构
279	
280	```python
281	# types.py
282	from pydantic import BaseModel
283	
284	class PoolEntry(BaseModel):
285	    """候选池中的单个 API 条目。
286	
287	    weight 为浮点数。浮点比较使用误差容限 1e-9。
288	    """
289	    api_id: str              # API 唯一标识，如 "anthropic/claude-opus-4-6"
290	    weight: float = 1.0      # 相对概率权重，>= 0
291	    enabled: bool = True     # False = 不参与选择
292	
293	class TaskPool(BaseModel):
294	    """某个任务类型的 API 候选池（运行时结构）。
295	
296	    继承关系只在 TASK_HIERARCHY 中定义，
297	    TaskPool 本身不携带 inherit_from，避免双源头不一致。
298	    """
299	    task_type: str
300	    entries: list[PoolEntry]
301	
302	class PoolConfig(BaseModel):
303	    """候选池持久化配置。
304	
305	    只存储有独立配置的池（entries 非空）。
306	    未出现的 task_type 走继承链解析。
307	    """
308	    version: int = 1               # 配置格式版本，便于未来迁移
309	    default_weight: float = 1.0    # 新 API 加入时的默认权重
310	    pools: dict[str, list[PoolEntry]]  # task_type -> entries
311	```

313	### APIPoolManager
314	
315	```python
316	# manager.py
317	import threading
318	from copy import deepcopy
319	
320	class APIPoolManager:
321	    """管理所有任务类型的 API 候选池。
322	
323	    实现 ModelSelector Protocol。
324	    线程安全：所有写操作持锁，select() 持锁。
325	    全局单例，由 ModelRegistry 持有引用。
326	    """
327	
328	    def __init__(
329	        self,
330	        config: PoolConfig | None = None,
331	        *,
332	        default_pool: TaskPool | None = None,
333	    ):
334	        self._lock = threading.RLock()
335	        self._pools: dict[str, TaskPool] = {}
336	        self._default_pool = default_pool
337	        if config:
338	            self.replace_config(config)
339	        else:
340	            self._init_defaults()
341	
342	    # --- 选择（对外核心接口）---
343	
344	    def select(self, task_type: str, override: str | None = None) -> str:
345	        """从任务候选池中按权重随机选择。override 优先。"""
346	        if override:
347	            return override
348	        with self._lock:
349	            pool = resolve_pool(task_type, self._pools, default_pool=self._default_pool)
350	            return weighted_select(pool.entries)
351	
352	    # --- 注册 ---
353	
354	    def register_api(
355	        self,
356	        api_id: str,
357	        *,
358	        add_to: list[str] | None = None,
359	        exclude_from: list[str] | None = None,
360	    ) -> None:
361	        """注册新 API。默认加入所有已知任务类型的池。"""
362	        if add_to is not None and exclude_from is not None:
363	            raise ValueError("add_to and exclude_from are mutually exclusive")
364	
365	        entry = PoolEntry(api_id=api_id, weight=self._default_weight)
366	
367	        with self._lock:
368	            if add_to is not None:
369	                targets = set(add_to)
370	            else:
371	                targets = set(self._list_all_task_types())
372	                if exclude_from is not None:
373	                    targets -= set(exclude_from)
374	            for task_type in targets:
375	                self._ensure_pool(task_type).entries.append(deepcopy(entry))
376	
377	    def unregister_api(self, api_id: str) -> None:
378	        """从所有池中移除该 API。"""
379	        with self._lock:
380	            for pool in self._pools.values():
381	                pool.entries = [e for e in pool.entries if e.api_id != api_id]
382	
383	    # --- 用户调整 ---
384	
385	    def adjust_weight(self, task_type: str, api_id: str, weight: float) -> None: ...
386	    def upvote(self, task_type: str, api_id: str, delta: float = 0.5) -> None: ...
387	    def downvote(self, task_type: str, api_id: str, delta: float = 0.5) -> None:
388	        """调低权重。最低为 0（不参与选择但仍在池中，可用 upvote 恢复）。"""
389	    def disable(self, task_type: str, api_id: str) -> None:
390	        """禁用（enabled=False，保留条目，可 enable 恢复）。"""
391	    def enable(self, task_type: str, api_id: str) -> None: ...
392	    def remove(self, task_type: str, api_id: str) -> None:
393	        """永久删除条目（不可恢复）。"""
394	
395	    # --- 池 fork ---
396	
397	    def fork_pool(self, task_type: str) -> TaskPool:
398	        """为子类型创建独立池（从父池深拷贝）。
399	
400	        例如 /pool fork skill/commit → 拷贝 subagent/skill 池到 skill/commit。
401	        之后对 skill/commit 的调整不影响 subagent/skill。
402	        """
403	        with self._lock:
404	            parent = resolve_pool(task_type, self._pools, default_pool=self._default_pool)
405	            new_pool = TaskPool(
406	                task_type=task_type,
407	                entries=deepcopy(parent.entries),
408	            )
409	            self._pools[task_type] = new_pool
410	            return new_pool
411	
412	    # --- 查询 ---
413	
414	    def get_pool(self, task_type: str) -> TaskPool: ...
415	    def list_task_types(self) -> list[str]: ...
416	    def list_apis(self, task_type: str) -> list[PoolEntry]: ...
417	
418	    # --- 持久化 ---
419	
420	    def to_config(self) -> PoolConfig:
421	        """导出当前池状态为可持久化配置。
422	
423	        只导出有独立池的 task_type（entries 非空）。
424	        调用时机：/pool save、会话结束时 config 模块调用。
425	        """
426	    def replace_config(self, config: PoolConfig) -> None:
427	        """用配置文件内容替换当前池状态（加载配置时调用）。"""
428	
429	    # --- 内部 ---
430	
431	    @property
432	    def _default_weight(self) -> float: ...
433	    def _list_all_task_types(self) -> set[str]:
434	        """所有已知任务类型（TASK_HIERARCHY + 已有独立池）。"""
435	    def _ensure_pool(self, task_type: str) -> TaskPool: ...
436	    def _init_defaults(self) -> None:
437	        """初始化根任务类型空池。"""
438	```

440	### 选择算法
441	
442	```python
443	# selector.py
444	import random
445	import math
446	
447	WEIGHT_EPSILON = 1e-9  # 权重比较容限
448	
449	class NoAPIAvailable(Exception):
450	    """池中无可用 API。"""
451	
452	def weighted_select(entries: Sequence[PoolEntry]) -> str:
453	    """按权重随机选择一个 API（纯函数）。
454	
455	    权重 <= EPSILON 或 disabled 的条目不参与选择。
456	    """
457	    active = [e for e in entries if e.enabled and e.weight > WEIGHT_EPSILON]
458	    if not active:
459	        raise NoAPIAvailable("no active API in pool")
460	
461	    total = sum(e.weight for e in active)
462	    r = random.uniform(0, total)
463	    cumulative = 0.0
464	    for e in active:
465	        cumulative += e.weight
466	        if r <= cumulative:
467	            return e.api_id
468	    # 浮点舍入导致未命中，返回最后一个
469	    return active[-1].api_id
470	```

472	### 任务类型继承（单一真相源）
473	
474	```python
475	# tasks.py
476	
477	# 静态继承关系。继承关系只在这里定义，TaskPool 上不重复。
478	# 不在此处的 task_type：
479	#   - "skill/<name>" → resolve_parent 返回 "subagent/skill"
480	#   - 其他未知 → resolve_parent 返回 None（视为根类型）
481	TASK_HIERARCHY: dict[str, str | None] = {
482	    "main": None,
483	    "subagent": None,
484	    "continuous": None,
485	    "background": None,
486	    "subagent/explore": "subagent",
487	    "subagent/plan": "subagent",
488	    "subagent/general": "subagent",
489	    "subagent/compact": "subagent",
490	    "subagent/memory": "subagent",
491	    "subagent/skill": "subagent",
492	    "subagent/meta": "subagent",
493	    "subagent/classify": "subagent",
494	    "subagent/code": "subagent",
495	    "continuous/cron": "continuous",
496	    "continuous/monitor": "continuous",
497	    "continuous/heartbeat": "continuous",
498	    "background/dream": "continuous",
499	    "background/title": "continuous",
500	    "background/compact": "continuous",
501	    "background/flush": "continuous",
502	}
503	
504	def resolve_parent(task_type: str) -> str | None:
505	    """解析任务类型的父类型（纯函数）。
506	
507	    查找顺序：静态 TASK_HIERARCHY → skill/* 前缀 → 未知返回 None
508	    """
509	    if task_type in TASK_HIERARCHY:
510	        return TASK_HIERARCHY[task_type]
511	    if task_type.startswith("skill/"):
512	        return "subagent/skill"
513	    return None
514	
515	def resolve_pool(
516	    task_type: str,
517	    pools: dict[str, TaskPool],
518	    *,
519	    default_pool: TaskPool | None = None,
520	) -> TaskPool:
521	    """解析任务对应的最终池（纯函数）。
522	
523	    沿继承链向上查找，返回第一个有独立配置的池。
524	    整条链都没有独立池时，使用 default_pool。
525	    default_pool 也为 None 时抛出 NoAPIAvailable。
526	    """
527	    current = task_type
528	    while current is not None:
529	        pool = pools.get(current)
530	        if pool is not None and pool.entries:
531	            return pool
532	        current = resolve_parent(current)
533	    if default_pool is not None:
534	        return default_pool
535	    raise NoAPIAvailable(
536	        f"no pool for task_type '{task_type}' and no default pool configured"
537	    )
538	```
539	
540	**fallback 链示例**：`skill/commit` → `subagent/skill`（无独立池）→ `subagent`（有独立池）→ 使用 subagent 池。如果 `subagent` 也没池，回退到 `default_pool`。连 `default_pool` 都没有才抛异常。
541	
542	**未知 task_type**：用户/插件可能创建未注册的自定义 task_type。此时 `resolve_parent()` 返回 `None`，`resolve_pool()` 直接走到 `default_pool`。不会崩。
543	
544	### 持久化时机
545	
546	运行时调整（upvote/downvote/disable/fork）**只修改内存**，不自动落盘：
547	
548	- **会话结束时**：config 模块调用 `pool_manager.to_config()` 写入 `~/.wings/config.toml`
549	- **用户主动保存**：`/pool save` 命令
550	- **加载时**：config 模块读 TOML → PoolConfig → `pool_manager.replace_config()`
551	
552	这样避免每次 upvote 都写盘，也允许用户在一个会话中随意试，不满意不保存。

609	
610	参考 opensquilla 的装饰器模式：
611	
612	```python
613	# decorator.py
614	import functools
615	from typing import Type, get_type_hints
616	from pydantic import BaseModel
617	
618	def tool(
619	    *,
620	    name: str,
621	    description: str,
622	    search_hint: str,
623	    read_only: bool = False,
624	    destructive: bool = False,
625	):
626	    """将 async 函数注册为 Tool"""
627	    def decorator(fn):
628	        hints = get_type_hints(fn)
629	        # 第一个参数（input 之外）是 Pydantic model → input_schema
630	        input_type = next(
631	            (t for n, t in hints.items() if n != "return" and issubclass(t, BaseModel)),
632	            None,
633	        )
634	
635	        @functools.wraps(fn)
636	        class _ToolAdapter:
637	            name = name
638	            description = description
639	            search_hint = search_hint
640	
641	            def input_schema(self) -> dict:
642	                return input_type.model_json_schema() if input_type else {}
643	
644	            async def call(self, input, context):
645	                result = await fn(input, context)
646	                return ToolResult(output=str(result))
647	
648	            def is_enabled(self) -> bool:
649	                return True
650	
651	            def is_read_only(self, input=None) -> bool:
652	                return read_only
653	
654	            def is_destructive(self, input=None) -> bool:
655	                return destructive
656	
657	            def render_result(self, result: ToolResult) -> str:
658	                return result.output
659	
660	            def activity_description(self, input=None) -> str:
661	                return f"{name}..."
662	
663	        return _ToolAdapter()
664	    return decorator
665	```

786	### 模型转交 (Model Handoff)
787	
788	主对话中，候选池可能在同一会话的不同 turn 选出不同的模型。当同一模型的两次调用之间有其他模型被调用时，需要注入转交提示，让当前模型了解中间发生了什么，并审查是否需要修正。
789	
790	**触发条件**：模型 A 被调用 → 中间有其他模型被调用 → 模型 A 再次被调用。
791	
792	**注入的提示包含**：
793	1. 说明中间任务已转交给其他模型处理
794	2. 要求模型在进行当前任务之外，分析最近行为中是否有需要修正但尚未修正的问题
795	
796	```python
797	# handoff.py
798	from datetime import datetime
799	from pydantic import BaseModel
800	
801	class TurnRecord(BaseModel):
802	    """记录每个 turn 的模型调用情况"""
803	    turn_id: int
804	    model_id: str                                  # 使用的 API id
805	    timestamp: datetime
806	    user_input_summary: str                        # 用户输入摘要（一句话）
807	    tool_calls: list[str]                          # 该 turn 中调用的工具名
808	    summary: str = ""                              # 该 turn 做了什么（一句话）
809	
810	class HandoffDetector:
811	    """检测模型转交，生成转交提示。
812	
813	    在主对话的 AgentLoop 中维护一个 TurnRecord 列表。
814	    """
815	
816	    def detect(
817	        self,
818	        current_model: str,
819	        turn_history: list[TurnRecord],
820	    ) -> str | None:
821	        """检测是否需要注入转交提示。
822	
823	        返回转交提示文本，或 None（不需要）。
824	        """
825	        if len(turn_history) < 2:
826	            return None
827	
828	        # 找到当前模型上一次出现的 turn
829	        previous_same_model = None
830	        has_other_between = False
831	        for turn in reversed(turn_history):
832	            if turn.model_id == current_model:
833	                previous_same_model = turn
834	                break
835	
836	        if previous_same_model is None:
837	            return None  # 该模型首次出现，无需转交
838	
839	        # 检查中间是否有其他模型
840	        saw_self = False
841	        intermediate_turns: list[TurnRecord] = []
842	        for turn in reversed(turn_history):
843	            if turn is previous_same_model:
844	                saw_self = True
845	            elif saw_self and turn.model_id != current_model:
846	                intermediate_turns.append(turn)
847	
848	        if not intermediate_turns:
849	            return None  # 中间没有其他模型
850	
851	        return self._build_handoff_prompt(
852	            current_model=current_model,
853	            previous_turn=previous_same_model,
854	            intermediate_turns=intermediate_turns,
855	        )
856	
857	    def _build_handoff_prompt(
858	        self,
859	        current_model: str,
860	        previous_turn: TurnRecord,
861	        intermediate_turns: list[TurnRecord],
862	    ) -> str:
863	        """构建转交提示。"""
864	        other_models = sorted({t.model_id for t in intermediate_turns})
865	        turns_desc = "\n".join(
866	            f"  - [{t.model_id}] {t.summary or t.user_input_summary}"
867	            for t in reversed(intermediate_turns)
868	        )
869	
870	        return f"""\
871	[系统提示] 自你上次参与此对话（turn #{previous_turn.turn_id}）以来，共有 \
872	{len(intermediate_turns)} 个 turn 转交给了其他模型处理：{', '.join(other_models)}。
873	
874	中间发生的工作:
875	{turns_desc}
876	
877	在进行当前任务之前，请：
878	1. 审查这些中间 turn 的行为是否有需要修正但尚未修正的问题（如不一致的代码风格、
879	   冲突的决策、遗漏的边界条件等）
880	2. 如果发现问题，优先修正后再进行当前任务
881	3. 如果没有需要修正的问题，直接进行当前任务"""
882	```

884	### loop.py（含转交检测）
885	
886	```python
887	# loop.py
888	from typing import AsyncIterator
889	from wings.messages.types import Message, StreamEvent, Role
890	from wings.tools.base import Tool, ToolContext, ToolResult
891	from wings.query.engine import QueryEngine
892	from wings.models.protocol import ModelConfig
893	from wings.permissions.pipeline import PermissionPipeline
894	from wings.routing.protocol import ModelSelector
895	
896	class AgentLoop:
897	    def __init__(
898	        self,
899	        query_engine: QueryEngine,
900	        tool_registry: ToolRegistry,
901	        permission_pipeline: PermissionPipeline,
902	        model_selector: ModelSelector,
903	    ):
904	        self._turn_history: list[TurnRecord] = []
905	        self._handoff_detector = HandoffDetector()
906	
907	    async def run(
908	        self,
909	        user_input: str,
910	        context: AgentContext,
911	    ) -> AsyncIterator[StreamEvent]:
912	        messages = self._assemble_messages(user_input, context)
913	        model = self._select_model(context)  # from pool by task_type
914	        config = self._build_config(model)
915	        tools = self._tool_registry.get_schemas()
916	
917	        # --- 模型转交检测 ---
918	        if context.task_type == "main":
919	            handoff_prompt = self._handoff_detector.detect(
920	                model, self._turn_history,
921	            )
922	            if handoff_prompt:
923	                # 作为 system 消息注入到消息列表
924	                messages.append(Message(role=Role.USER, content=[TextBlock(text=handoff_prompt)]))
925	
926	        # 记录当前 turn
927	        self._turn_history.append(TurnRecord(
928	            turn_id=len(self._turn_history),
929	            model_id=model,
930	            timestamp=datetime.now(),
931	            user_input_summary=user_input[:200],
932	            tool_calls=[],
933	        ))
934	
935	        while True:
936	            had_tool_use = False
937	
938	            async for event in self._query_engine.stream(
939	                messages, model, tools, config,
940	            ):
941	                if event.type == "tool_use":
942	                    tool = self._tool_registry.get(event.name)
943	                    if tool:
944	                        decision = await self._permission_pipeline.check(
945	                            tool, event.input, self._tool_context,
946	                        )
947	                        if decision == "deny":
948	                            messages.append(self._inject_error(event.id, "Permission denied"))
949	                            had_tool_use = True
950	                            break
951	
952	                        # 记录工具调用
953	                        self._turn_history[-1].tool_calls.append(event.name)
954	
955	                        result = await tool.call(event.input, self._tool_context)
956	                        messages.append(ToolResultMessage(
957	                            role=Role.USER,
958	                            content=[ToolResultBlock(
959	                                tool_use_id=event.id,
960	                                content=result.output,
961	                                is_error=result.error is not None,
962	                            )],
963	                        ))
964	                        had_tool_use = True
965	                        break
966	
967	                yield event  # 文本、thinking 直接输出
968	
969	            if not had_tool_use:
970	                # 更新 turn 摘要
971	                self._turn_history[-1].summary = self._extract_summary(messages)
972	                return  # end_turn
973	
974	    def _assemble_messages(self, user_input: str, context: AgentContext) -> list[Message]: ...
975	    def _select_model(self, context: AgentContext) -> str:
976	        """从 ModelSelector 获取 API。"""
977	        return self._selector.select(
978	            task_type=context.task_type,
979	            override=context.model_override,
980	        )
981	    def _inject_error(self, tool_use_id: str, error: str) -> Message: ...
982	    def _extract_summary(self, messages: list[Message]) -> str: ...
983	```

986	### 子 Agent 类型与任务分类
987	
988	参考 claude-code 和 opensquilla 的子 agent 体系，wings 将所有任务分为以下类型，每种类型对应独立的 API 候选池，可选用不同的默认工具集。
989	
990	#### 任务类型全景
991	
992	| 任务类型 | 触发场景 | 来源参考 | 默认 API 池 | 默认工具集 |
993	|----------|----------|----------|-------------|-----------|
994	| `main` | 主 session 与用户对话 | 两个项目的主 REPL 循环 | 全部 API | 全部工具 |
995	| `subagent/explore` | 代码库探索、搜索、理解结构 | claude-code `exploreAgent` | 继承 subagent | read, glob, grep, web_search |
996	| `subagent/plan` | 方案规划、架构设计 | claude-code `planAgent` | 继承 subagent | read, glob, grep, web_search |
997	| `subagent/general` | 通用子任务委托 | claude-code `generalPurposeAgent` | subagent 池 | 受限工具集 |
998	| `subagent/compact` | 对话压缩/摘要 | 两个项目都有 compact 服务 | subagent 池 | 无工具（纯 LLM 摘要） |
999	| `subagent/memory` | 记忆保存、刷新、巩固 (Dream) | opensquilla memory flush/dream | subagent 池 | memory 读写工具 |
1000	| `subagent/skill` | 技能执行（fork 模式，未细分时回退到此） | 两个项目的 skill 系统 | subagent 池 | 技能指定的工具 |
1001	| `skill/<name>` | 特定技能（如 `skill/commit`, `skill/review-pr`）| 继承 subagent/skill | 继承 subagent/skill | 该技能指定的工具 |
1002	| `subagent/meta` | DAG 多步编排子任务 | opensquilla meta-skill orchestrator | subagent 池 | 编排器指定工具 |
1003	| `subagent/classify` | 确定性分类/路由决策 | opensquilla `llm_classify` | subagent 池 | 无工具 |
1004	| `subagent/chat` | 隔离对话（如敏感内容处理） | opensquilla `llm_chat` | subagent 池 | 受限工具 |
1005	| `subagent/code` | 编码模式子任务 | opensquilla coding_mode | subagent 池 | read, write, edit, bash |
1006	| `continuous/cron` | 定时任务执行 | claude-code CronTool, opensquilla scheduler | continuous 池 | 任务指定工具 |
1007	| `continuous/monitor` | 轮询/监控 | claude-code MonitorTool | continuous 池 | 状态检查工具 |
1008	| `continuous/heartbeat` | 心跳检查 | opensquilla HEARTBEAT.md | continuous 池 | 无工具（轻量检查） |
1009	| `background/dream` | 离线记忆巩固 | 两个项目的 Dream 系统 | continuous 池 | memory 读写 |
1010	| `background/title` | 会话命名 | opensquilla session naming | continuous 池 | 无工具 |
1011	| `background/compact` | 后台上下文压缩 | claude-code auto-compact | continuous 池 | 无工具 |
1012	| `background/flush` | 会话关闭记忆刷新 | opensquilla SessionFlushService | continuous 池 | memory 写入 |
1013	
1014	#### 详细说明
1015	
1016	**Explore Agent** (claude-code `exploreAgent`)
1017	- 触发：模型需要理解不熟悉的代码库结构时
1018	- 特点：只读工具为主，fork 语义隔离上下文
1019	- System prompt 强调"快速搜索、不修改文件"
1020	- API 池偏好：便宜快速的模型（探索不需要深度推理）
1021	
1022	**Plan Agent** (claude-code `planAgent`)
1023	- 触发：需要设计方案、规划实施步骤时
1024	- 特点：可读但不应修改文件
1025	- System prompt 强调"考虑架构权衡、设计可验证的步骤"
1026	- API 池偏好：推理能力强的模型
1027	
1028	**General Purpose Agent** (claude-code `generalPurposeAgent`)
1029	- 触发：通用子任务委托，未匹配到专门类型时回退
1030	- 特点：受限但完整的工具集
1031	- 用于隔离上下文、并行执行独立子任务
1032	
1033	**Compact Agent**
1034	- 触发：上下文窗口接近上限，需压缩历史消息
1035	- claude-code 有 4 种 compact prompt（完整/部分/前缀/无工具）
1036	- opensquilla 额外提取"义务"（目标、约束、决策、工件、错误）
1037	- 纯 LLM 摘要任务，不需要工具
1038	
1039	**Memory Agent**
1040	- 触发：会话结束、定期保存记忆、Dream 巩固
1041	- opensquilla 的 Dream 是离线 LLM 过程：扫描新文件 → 证据评分 → 排序 → LLM 决策 upsert/merge/skip → 应用到 MEMORY.md
1042	- claude-code 在会话结束时自动保存学习内容
1043	
1044	**Meta Agent** (opensquilla meta-skill orchestrator)
1045	- 触发：复杂多步技能需要分解为 DAG 执行时
1046	- SOP 编译器将自然语言工作流编译为 DAG
1047	- 每个节点作为独立子 agent 执行，支持并行和依赖
1048	
1049	**Cron / Monitor / Heartbeat** (continuous 类)
1050	- 定时任务：用户设定的周期性任务（如"每 5 分钟检查部署状态"）
1051	- 监控：后台轮询检查状态变化
1052	- 心跳：opensquilla 的心跳机制，定期注入 HEARTBEAT.md 内容
1053	
1054	**Skill Agent** (per-skill 独立池)
1055	- 每个 skill 如果触发 API 调用，都可以有自己的候选池 `skill/<name>`
1056	- 例如 `/pool` 可以为 `skill/commit` 单独设置池（偏好快速模型），为 `skill/review-pr` 设置另一个池（偏好推理强模型）

1149	### pipeline.py
1150	
1151	```python
1152	# pipeline.py
1153	from typing import Literal
1154	from wings.tools.base import Tool, ToolContext
1155	
1156	PermissionResult = Literal["allow", "deny", "ask"]
1157	
1158	class PermissionPipeline:
1159	    def __init__(
1160	        self,
1161	        rules: "PermissionRules",
1162	        hook_runner: "HookRunner | None" = None,
1163	    ): ...
1164	
1165	    async def check(
1166	        self,
1167	        tool: Tool,
1168	        tool_input: object,
1169	        context: ToolContext,
1170	    ) -> PermissionResult:
1171	        # Stage 1: 静态规则
1172	        static_result = self._rules.match(tool.name)
1173	        if static_result != "ask":
1174	            return static_result
1175	
1176	        # Stage 2: 自动分类（只读操作自动放行）
1177	        if tool.is_read_only(tool_input):
1178	            return "allow"
1179	
1180	        # Stage 3: hooks
1181	        if self._hook_runner:
1182	            hook_result = await self._hook_runner.run_pre_tool_use(
1183	                tool.name, tool_input,
1184	            )
1185	            if hook_result is not None:
1186	                return hook_result
1187	
1188	        # Stage 4: 交互式审批（交给 UI）
1189	        return "ask"
1190	```

1192	### rules.py
1193	
1194	```python
1195	# rules.py
1196	class PermissionRules:
1197	    allowlist: set[str]     # 始终允许
1198	    denylist: set[str]      # 始终拒绝
1199	    asklist: set[str]       # 每次询问
1200	
1201	    @classmethod
1202	    def from_config(cls, config: dict) -> "PermissionRules": ...
1203	
1204	    def match(self, tool_name: str) -> PermissionResult:
1205	        if tool_name in self.denylist:
1206	            return "deny"
1207	        if tool_name in self.allowlist:
1208	            return "allow"
1209	        return "ask"
1210	
1211	    def add_allow(self, tool_name: str) -> None: ...
1212	    def add_deny(self, tool_name: str) -> None: ...

1496	### 开发过程中的设计决策与反思
1497	
1498	#### 1. MessageNormalizer 的补位
1499	
1500	设计文档定义了带 `to_internal(provider, messages)` / `to_provider(provider, messages)` 的 Protocol，但 Phase 1 实现只有裸函数（`from_anthropic`, `to_openai` 等）。Phase 2 开发时发现：QueryEngine 需要一个统一入口来路由到不同 provider 的转换器，否则每个调用方都要写 if-else 分支。
1501	
1502	**决策**: 在 Phase 1 中追加 `MessageNormalizer` 类，内部维护 `_from_provider` / `_to_provider` 注册表，模块加载时自动注册 anthropic 和 openai。对外暴露模块级 `normalizer` 单例。代价很小（~50 行），消除了未来的 if-else 扩散。
1503	
1504	#### 2. Protocol 作为模块边界
1505	
1506	routing 和 models 两个模块都遵循同一个模式：定义 Protocol → 实现类 → 调用方只依赖 Protocol。
1507	
1508	- `ModelSelector` Protocol → `APIPoolManager` 实现 → `ModelRegistry` / `AgentLoop` 依赖
1509	- `ModelProvider` Protocol → `AnthropicProvider` / `OpenAIProvider` 实现 → `QueryEngine` 依赖
1510	
1511	**效果**: 换池实现（比如从加权随机换成 epsilon-greedy）不需要改 ModelRegistry。换 provider（比如新增 Google）不需要改 QueryEngine。Protocol 是零成本的类型安全抽象——Python 不做运行时检查，但 mypy 能做。
1512	
1513	#### 3. 纯函数层的价值
1514	
1515	routing 模块将 `weighted_select()`、`resolve_parent()`、`resolve_pool()` 作为纯函数独立出来。它们在测试中不需要构造 Manager、不需要 mock 锁、不需要关心线程安全。47 个 routing 测试中，~30 个只测纯函数。
1516	
1517	**决策**: 保持这个模式。Phase 3 的 tools、Phase 5 的 permissions 也应该先写纯函数层。
1518	
1519	#### 4. 设计文档 vs 实现的差异
1520	
1521	部分设计细节在实现时调整了：
1522	
1523	| 设计 | 实现 | 原因 |
1524	|------|------|------|
1525	| `TaskPool.inherit_from` 字段 | 删除，只用 `TASK_HIERARCHY` | 双源头会不一致 |

1572	#### 7. @tool 装饰器的 Python 闭包陷阱
1573	
1574	设计文档中示意了简化的装饰器实现：
1575	
1576	```python
1577	class _ToolAdapter:
1578	    name = name  # 错误：类体无法访问外层函数变量
1579	```
1580	
1581	Python class body 有自己的隔离作用域，不能直接引用外层 `def tool()` 的参数。实现时改为：先将参数赋给局部变量（`_name = name`），方法通过闭包引用这些局部变量，类属性在类体定义后单独赋值。
1582	
1583	**教训**：设计文档中的 Python 代码是伪代码，部分写法无法直接执行。后续设计评审应该关注语言特性边界。

1636	#### 11. 消息双重转换 bug（Phase 4 发现并修复）
1637	
1638	最初 `QueryEngine` 持有 `MessageNormalizer`，在调用 provider 之前将 `list[Message]` 转为 `list[dict]`。但 `AnthropicProvider` / `OpenAIProvider` 内部也会调用 `to_anthropic()` / `to_openai_messages()` 做同样的转换。dict 被当成 Message 传入 adapter 会直接 crash。
1639	
1640	**根因**: 测试使用 mock provider（不检查消息格式），144 个测试全绿，但真实调用会炸。
1641	
1642	**修复**: 去掉 engine 中的消息转换和 normalizer 依赖。Provider 自己负责格式转换——engine 只做错误处理和重试。
1643	
1644	**教训**: mock 测试覆盖了"控制流"但漏掉了"数据流"。关键数据路径（消息格式的端到端传递）需要至少一个集成测试用真实 adapter 跑一遍。
1645	
1646	#### 12. QueryEngine 的职责收窄
1647	
1648	修复后 `QueryEngine` 只做两件事：(1) 从 registry 找 provider，(2) 带指数退避重试调用。消息转换、模型选择、工具执行都不归它管。
1649	
1650	好处：Phase 6 AgentLoop 的职责划分更清晰——AgentLoop 负责循环和工具调度，QueryEngine 只负责"调用一次 LLM"。
1651	
1652	#### 13. HandoffDetector 逆向遍历逻辑（Phase 6a 发现并修复）
1653	
1654	`HandoffDetector` 用 `reversed(turn_history)` 从最新到最旧遍历，寻找当前模型上次出现的位置，收集此期间的"中间 turn"。最初用 `saw_self` 标志位：
1655	
1656	```python
1657	# 错误版本
1658	if turn is previous_turn:
1659	    saw_self = True
1660	elif saw_self and turn.model_id != current_model:
1661	    intermediate.append(turn)
1662	```
1663	
1664	逆向遍历时，中间 turn 出现在上次 self-turn **之前**（还没到达 self-turn），`saw_self` 永远为 False。正确逻辑是：碰到 self-turn 直接 break，之前遇到的不同模型 turn 都收集：
1665	
1666	```python
1667	# 修复后
1668	if turn is previous_turn:
1669	    break
1670	if turn.model_id != current_model:
1671	    intermediate.append(turn)
1672	```
1673	
1674	**教训**: 正向/逆向遍历的标志位语义相反。写遍历逻辑时应该在测试里覆盖多个 turn 的排列组合（3+ turn 的场景）。
1675	
1676	#### 14. AgentLoop 先用 chat() 而非 stream()
1677	
1678	设计文档中 AgentLoop 用 `stream()` 获取流式事件。但 tool_use 的完整 input 在 streaming 模式下需要跨越多个 `ToolUseDelta` 累积 `partial_json`。当前 provider 只产出 delta 事件，不产出完整的 `ToolUseBlock`。
1679	
1680	**决策**: Phase 6a 用 `chat()`（非流式）实现完整循环——tool_use block 是完整的，可立即执行。流式模式等 provider 的 `stream()` 能产出完整 tool_use 后再接，循环本身与模式无关。
1681	
1682	#### 15. Per-call 模型选择（Phase 6a 后续改进）
1683	
1684	设计文档中模型选择每个 turn 一次。但 wings 的核心理念是"每次模型调用都从候选池随机选择"。将 `model = self._select_model()` 移入 `while True` 循环内：首轮 API 调用和工具执行后的每次后续调用都独立选模型。handoff 检测和 turn record 在第一次循环时执行。
1685	
1686	#### 16. Anthropic 消息格式：tool_result 必须分组（Phase 6a bug fix）
1687	
1688	Anthropic 协议要求一个 assistant 消息中的所有 tool_use，其对应的 tool_result 必须归组在紧接着的同一个 user 消息中。之前为每个 tool_result 单独建 Message，DeepSeek API 报 "tool_use ids without tool_result"。修复为先收集所有结果，再一次 append。
1689	
1690	**教训**：API 格式差异不仅是字段名不同，消息层面的结构约束（归组要求）更隐蔽。
1691	
1692	#### 17. @tool 装饰器：dict → Pydantic 自动转换（Phase 6a bug fix）
1693	
1694	`@tool` 装饰的 `call()` 方法收到的是 dict（来自 ToolUseBlock），但装饰的函数期望 Pydantic 对象。`input.timeout` 在 dict 上直接抛 AttributeError。修复：在 `call()` 中检测 dict 输入并调用 `_input_type(**input)` 转换。这之前被 mock 测试隐藏了。
